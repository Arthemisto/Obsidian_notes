**Коротко:** `sudo` запускает **одну команду** с правами **root**. Кто и что может запускать через `sudo`, задаётся правилами в **`/etc/sudoers`**.

---
### ⚙️ Что делает `sudo`
- Выполняет команду с привилегиями другого пользователя (обычно **root**).
- Эффект **временный**: права “повышаются” только на конкретную команду.
---
### 👤 Users & Groups: базовые команды
**Создать пользователя:**
sudo adduser username
**Создать группу:**
sudo addgroup groupname
**Добавить пользователя в группу:**
sudo usermod -aG groupname username
**Проверить членство:**
groups username

---
### 🚫 Почему плохо постоянно использовать `sudo`
- **Ownership mess:** файлы/папки создаются как `root:root` → потом обычный пользователь не может редактировать/удалять.
- **Security risk:** любая ошибка с `sudo` имеет максимальные последствия.
- **Плохая привычка:** пропадает понимание, где реально нужны админ-права.
✅ Практика: делай “обычную” работу от своего пользователя, а `sudo` — только для системных действий (установка пакетов, правка `/etc`, управление сервисами).
