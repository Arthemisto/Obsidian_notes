## 🪝 Mounting: подключение файловой системы к директории
**Коротко:** **mount** = “прикрепить” файловую систему/раздел (**filesystem/partition**) к папке (**mount point**) в дереве Linux. После монтирования содержимое становится доступным по пути этой папки.

---
## 1) 🧩 Что значит “смонтировать”
- Файловая система подключается к выбранной директории.
- С этого момента эта директория “показывает” содержимое подключённого раздела.
- Это интегрирует разные разделы/диски в **одно дерево каталогов**.
---
## 2) 🧱 Как сделать разные директории с разными FS
### Шаги (логика)
1. **Identify FS**: решить, какие FS нужны (например: `ext4`, `xfs`, `btrfs`, `ntfs`).
2. **Partition + format**: подготовить устройство (разделить и отформатировать) через `fdisk/parted/mkfs`.
3. **Create mount points**: создать папки, куда будем монтировать (например `/mnt/data`, `/mnt/backup`).
---
## 3) 🛠️ Мини-пример (ext4 + ntfs)
```
sudo mkdir /mnt/data  
sudo mkdir /mnt/backup  
  
sudo mount -t ext4 /dev/sda1 /mnt/data  
sudo mount -t ntfs /dev/sdb1 /mnt/backup  
  
df -h
```
- `mount -t <type> <device> <mount_point>` — подключение FS
- `df -h` — проверка, что смонтировалось и сколько места
---
## 4) 🧾 Идея для “закрепления” монтирования
Строки формата:
- `/dev/sda1 /mnt/data ext4 defaults 0 2`
- `/dev/sdb1 /mnt/backup ntfs defaults 0 2`
Это выглядит как записи для **`/etc/fstab`** (автомонтирование при загрузке).
