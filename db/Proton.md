**Коротко:** **Proton** — встроенный в **Steam** слой совместимости, который позволяет запускать Windows-игры на Linux: он “переводит” Windows API в понятный Linux формат и делает это без виртуальной машины, поэтому железо (GPU/CPU) используется напрямую и производительность часто близка к нативной.

---
## 🧩 Что такое Proton
- **Proton = compatibility layer** внутри Steam.
- Задача: запуск **Windows games** на **Linux** без отдельной установки Windows.
---
## 🔁 Как работает (идея без деталей)
- Игра ожидает **Windows APIs** → Proton выполняет **translation** этих вызовов в “Linux-readable” вид.
- Поэтому приложение ведёт себя “как на Windows”, но работает в Linux-среде.
---
## 🆚 Почему это не “обычная эмуляция”
Традиционный подход “эмулятор/VM”:
- запускаешь Windows в виртуалке
- игра работает “внутри” гостевой ОС
- накладные расходы выше, интеграция с GPU сложнее
Proton-подход:
- **не** требует полноценной Windows VM
- использует ресурсы ПК **напрямую**
- **better hardware utilization** → часто **comparable performance** к native build

---
## 🎮 Почему Proton реально важен
- Открывает доступ к огромной библиотеке Windows-игр на Linux.
- Делает Linux более “desktop/gaming-friendly”.
---
## ✅ Практика (что делать пользователю)
- В Steam: включить Proton (Compatibility settings) для игр, которые не имеют нативной версии.
- Проверять совместимость до установки через **ProtonDB**:
  - https://www.protondb.com/
---
## 🧭 Summary / Сводка
Proton — ключевая причина, почему “играть на Linux” стало реальным сценарием: это перевод Windows API без виртуалки, прямой доступ к железу и производительность, часто близкая к нативной.
